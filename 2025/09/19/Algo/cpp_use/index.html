因为auto &p:mp是对集合中每个元素的引用，后者是对每个元素的副本。使用前者可以避免不必要的拷贝开销，提升性能，尤其是当元素类型较大时，创建一个副本会消耗额外的内存和CPU周期。在大多数情况下，特别是处理std::string或自定义类对象时，推荐使用引用遍历以提高效率。

常用场景

用于快速查找、计数、映射关系。比如 判断数组是否包含重复元素、计数器、统计单词频率等。

3. unordered_set

std::unordered_set 是 C++ STL 提供的一个哈希集合（Hash Set）容器。它存储唯一的元素，且不保证元素的顺序。和 std::set 不同，unordered_set 的底层实现是哈希表（hash table），支持高效的插入、删除和查找操作，平均时间复杂度为 𝒪(1)。

常见操作

常用场景

用于快速查找唯一元素、去重、集合运算等。比如 判断数组是否包含重复元素、求两个数组的交集/并集等。

4. priority_queue

std::priority_queue 是 C++ STL 提供的一个优先队列容器。它是一种基于堆（heap）数据结构实现的容器，支持高效的插入和删除操作。默认情况下，priority_queue 是一个最大堆（max-heap），即每次访问或删除的都是当前队列中最大的元素，其插入和删除最大元素的时间复杂度为 𝒪(log n)。

常用场景

常用于需要频繁调取最大值/最小值的场景，如滑动窗口最大值、合并区间、top K 问题等。

5. queue

std::queue 是 C++ STL 提供的一个队列容器。它是一种先进先出（FIFO, First In First Out）的数据结构，支持在队列的末尾插入元素，在队列的前端删除元素。queue 底层通常是通过 deque（双端队列）或 list（链表）实现的。

6. deque

std::deque 是 C++ STL 提供的一个双端队列容器。它支持在队列的两端高效地插入和删除元素，同时也支持随机访问。deque 底层通常是通过一系列连续的内存块（chunk）实现的，因此在两端插入和删除元素的时间复杂度为 𝒪(1)，而随机访问的时间复杂度为 𝒪(1)。可以看作是queue和stack的结合体。

7. 链表

链表是一种常见的数据结构，由一系列节点组成，每个节点包含数据和指向下一个节点的指针。链表的主要优点是插入和删除操作高效，时间复杂度为 𝒪(1)，但随机访问元素的时间复杂度为 𝒪(n)。

8. 关于new和delete

C++中的内存管理

C++中的内存管理主要分为两种类型：栈内存（stack memory）和堆内存（heap memory）。

  • 栈内存：由编译器自动管理，函数调用时分配，函数返回时释放。适用于局部变量和函数参数，分配和释放速度快，但空间有限。
  • 堆内存：由程序员手动管理，使用new关键字分配，使用delete关键字释放。适用于需要动态分配内存的场景，空间较大，但分配和释放速度较慢，容易出现内存泄漏和碎片化问题。

  1. 动态数据结构（链表、树、图等），链表节点、树节点、图节点的数量通常运行时才能确定；
  2. 长期存在的对象，比如游戏中的一个玩家、后台服务器中的一个连接对象；
  3. 需要在函数之间共享数据，函数返回的数据需要在调用函数中使用。

9. 基本数字类型和浮点型数字类型

C++中常见的基本数字类型包括整数类型和浮点类型。它们在内存中的大小和表示范围各不相同。

char 1 byte -128 ~ 127 或 0 ~ 255（取决于是否有符号） CHAR_MIN, CHAR_MAX
signed char 1 byte -128 ~ 127 SCHAR_MIN, SCHAR_MAX
unsigned char 1 byte 0 ~ 255 UCHAR_MAX
short 2 bytes -32,768 ~ 32,767 SHRT_MIN, SHRT_MAX

10. 访问控制

C++中的访问控制用于限制类成员（属性和方法）的访问权限，主要有三种访问修饰符：public、protected 和 private。

  • public：公共成员，可以被类的任何对象访问，包括类外部的代码。
  • protected：受保护成员，可以被类的成员函数和派生类（子类）访问，但不能被类外部的代码访问。
  • private：私有成员，只能被类的成员函数访问，不能被派生类和类外部的代码访问。
